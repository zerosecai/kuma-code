Why Kuma is different

Most AI coding tools force a trade-off: power vs. privacy. Kuma refuses that trade.

🛡️

Local-first by default

Run agents on your local Ollama or LM Studio. Your code, your prompts, your context — none of it leaves your network unless you opt in.

$ kuma --local-only
🤖

Multi-agent in parallel — locally

Planner, Coder, and Reviewer agents work simultaneously on your task. Each picks the right model for its job. Architecture supports scaling to 10 agents.

3 agents → 10 agents
📦

Skill packs that punch above their weight

1GB modular skill packs make a 3B local model perform like a 70B-class model on TypeScript, React, and Vite. Two-level lookup keeps inference fast.

3B model → 70B output
🔄

Learning loop, anywhere

Your agents learn from your patterns whether you're running local Ollama or bursting to Ollama Cloud. Switch providers; the loop persists.

Local + Cloud

Built for teams that can't ship code to a third-party cloud

Kuma is designed for teams in regulated industries — finance, healthcare, defense, government — where source code can't leave the corporate network. A local-only configuration has zero required outbound connections.

  • Air-gapped operation with local Ollama / LM Studio
  • Open source code — auditable in a security review
  • No telemetry by default — verifiable in config.ts
  • Provider routing decisions surfaced in the UI before sending
  • SOC 2 controls and audit logging on the roadmap
